# Tree Structure

Reference for the viewer tree area in `viewer_html/`.

## Purpose

The tree panel is the left-side HDF5 structure browser.

It is responsible for:
- rendering groups and datasets
- lazy-loading children
- expand/collapse behavior
- tree selection
- retry after tree-load failure
- line-compare dataset add buttons

## Main Shell IDs

These IDs must exist in the shell file before JS boot.

Shell seed files:
- `viewer_html/index.html`
- `viewer_html/index.php`

Validation source:
- `viewer_html/js/core/domRefs.js`

| ID | Purpose | Main file using it |
|---|---|---|
| `viewer-sidebar` | tree container root | `js/views/viewerView.js` |
| `tree-panel` | tree section wrapper | `js/components/sidebarTree.js` |
| `tree-list` | list root for nodes | `js/components/sidebarTree.js` |
| `tree-status` | tree status message area | `js/views/viewerView.js` |
| `sidebar-close-btn` | close sidebar button | `js/views/viewerView.js` |

## Generated Tree Button Hooks

These are not seeded in `index.html`. They are generated by `renderSidebarTree()` in:
- `viewer_html/js/components/sidebarTree.js`

| Selector | Purpose | Action target |
|---|---|---|
| `[data-tree-select-path]` | select group or dataset | `selectTreeNode()` |
| `[data-tree-select-type]` | selected node type payload | `selectTreeNode()` |
| `[data-tree-select-name]` | selected node label payload | `selectTreeNode()` |
| `[data-tree-toggle-path]` | expand/collapse a group | `toggleTreePath()` |
| `[data-tree-retry-path]` | retry failed child load | `loadTreeChildren(..., { force: true })` |
| `[data-tree-compare-add-path]` | add dataset to line compare | `addLineCompareDataset()` |
| `[data-tree-compare-add-name]` | compare payload label | `addLineCompareDataset()` |
| `[data-tree-compare-add-type]` | compare payload node type | `addLineCompareDataset()` |
| `[data-tree-compare-add-dtype]` | compare payload dtype | `addLineCompareDataset()` |
| `[data-tree-compare-add-shape]` | compare payload shape | `addLineCompareDataset()` |
| `[data-tree-compare-add-ndim]` | compare payload ndim | `addLineCompareDataset()` |

## File Ownership

| File | Role |
|---|---|
| `viewer_html/js/components/sidebarTree.js` | tree HTML generation and delegated tree click handling |
| `viewer_html/js/state/reducers/treeActions.js` | expand, select, breadcrumb navigation, child loading |
| `viewer_html/js/state/reducers/filesActions.js` | initial `openViewer()` triggers root tree load |
| `viewer_html/js/api/hdf5Service.js` | backend calls for `getFileChildren()` |
| `viewer_html/js/views/viewerView.js` | injects rendered tree HTML into `#viewer-sidebar` and updates `#tree-status` |
| `viewer_html/css/viewer.css` | canonical tree layout and styling |
| `viewer_html/css/components/tree.css` | tree stylesheet entrypoint/shim |

## Render Flow

1. `app-viewer.js` boots the viewer.
2. `viewerView.js` calls `renderSidebarTree(state)`.
3. The returned markup is injected into `#viewer-sidebar`.
4. `bindSidebarTreeEvents()` listens on the shell root and reacts to the `data-tree-*` selectors.
5. `treeActions.js` updates state and triggers data loads.

## Change Notes

When changing tree structure, check these three places together:
- `js/core/domRefs.js`
- `js/components/sidebarTree.js`
- `js/state/reducers/treeActions.js`

If you rename an ID in the shell, update:
- the shell file (`index.html` or `index.php`)
- `js/core/domRefs.js`
- any renderer that emits the same ID again
